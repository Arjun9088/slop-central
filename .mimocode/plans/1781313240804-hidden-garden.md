# Expense Tracker Android App — Implementation Plan

## Overview
A greenfield Android expense tracker using Room (local DB) as the primary source of truth, with background sync to Google Sheets via WorkManager. Follows the same architecture as the sibling `article-vault` project: Kotlin + Jetpack Compose + Room + Hilt + MVVM + StateFlow.

## 3. Room Database

### Entity: `Expense`
| Column | Type | Notes |
|---|---|---|
| id | Long (auto PK) | Auto-generated |
| date | String | "YYYY-MM-DD" format |
| description | String | Free text |
| category | String | Enum name (FOOD_DRINK, SHOPPING, etc.) |
| amount | Double | |
| paymentMethod | String | Enum name (CASH, CREDIT_CARD, etc.) |
| createdAt | Long | Timestamp |
| syncedToSheet | Boolean | false = needs sync |
| sheetRowId | Long? | Row in Google Sheet (null if not synced) |

### Enums
- `ExpenseCategory`: FOOD_DRINK, SHOPPING, TRANSPORTATION, ENTERTAINMENT, BILLS_UTILITIES, OTHER
- `PaymentMethod`: CASH, CREDIT_CARD, DEBIT_CARD, DIGITAL_WALLET, GPAY

### DAO: `ExpenseDao`
- `insert()`, `update()`, `delete()`, `deleteById()`
- `observeAll()` → `Flow<List<Expense>>` (sorted by date DESC)
- `observeByDateRange()`, `observeByCategory()`
- `getUnsynced()` → suspend list for SyncWorker
- `markSynced(id, sheetRowId)`, `markUnsynced(id)`
- `getTotalInRange()`, `getTotalByCategoryInRange()`, `getTotalByPaymentMethodInRange()`
- `count()`

## 4. Google Sheets API Integration

### Auth: OAuth 2.0 via Google Sign-In
1. User taps "Connect Google Account" in Settings
2. `play-services-auth` launches Google Sign-In picker
3. Credential passed to `GoogleSheetsService` to build `Sheets` client

### Sheet Format
| A (Date) | B (Description) | C (Category) | D (Amount) | E (Payment Method) |
|---|---|---|---|---|

### Service Methods
- `appendRow(expense)` → returns row ID
- `updateRow(rowId, expense)` → update existing
- `deleteRow(rowId)` → delete from sheet
- `fetchAllRows()` → for future pull/import

### Spreadsheet ID
Stored in `SyncPreferences` (SharedPreferences). User configures once in Settings.

---

## 5. Sync Architecture

### Strategy: Offline-first, Room is source of truth
1. All writes go to Room first with `syncedToSheet = false`
2. `SyncWorker` (WorkManager) runs periodically (15 min) with `NetworkType.CONNECTED` constraint
3. Worker reads unsynced expenses, pushes each to Google Sheets
4. On success → `markSynced(id, sheetRowId)`
5. On failure → `Result.retry()` with exponential backoff
6. Sheets acts as a read-only mirror/backup (no bidirectional pull for now)

## 7. Quick-Entry Screen (EntryScreen)

### Layout
```
Date        → OutlinedTextField with today's date, tap opens DatePicker
Description → OutlinedTextField
Category    → FlowRow of FilterChips (6 predefined categories)
Amount      → OutlinedTextField, KeyboardType.Decimal, AUTO-FOCUSED
Payment     → FlowRow of FilterChips (5 payment methods)
[Save]      → Full-width FilledButton
```

### Key UX Decisions
- Amount field auto-focused on screen open (minimize taps)
- Date defaults to today (YYYY-MM-DD) but editable via DatePicker
- Category defaults to first option (Food and drink) — one tap to change
- Payment defaults to Cash — one tap to change
- On save → pop back to list immediately
- Total flow: tap FAB → type amount → tap category chip → tap payment chip → tap Save = ~4 taps

### ViewModel State
- `EntryUiState` data class with all fields, `isSaving`, `saveSuccess`, `errorMessage`
- Validation: amount must be > 0, description must not be blank
- On save success: `onBack()` navigation callback

## 8. Data Flow

```
UI (EntryScreen) → ViewModel.save() → ExpenseDao.insert() → Room DB
                                                    ↓
WorkManager SyncWorker (15 min, network required)
                                                    ↓
SyncWorker → getUnsynced() → GoogleSheetsService.appendRow() → Google Sheets
                                                    ↓
SyncWorker → markSynced(id, rowId) → Room DB updated
                                                    ↓
ExpenseListScreen re-collects Flow → UI updates with sync indicator
```

---

## 9. Error Handling

- **Room errors**: Caught in ViewModel, shown via `errorMessage` → Snackbar
- **Sync errors**: `Result.retry()` with exponential backoff via WorkManager
- **Auth errors (401/403)**: User re-authenticates in Settings
- **No internet**: SyncWorker constraints prevent execution; expenses stay local

---

## 10. Implementation Order

1. **Gradle scaffolding** + AndroidManifest → `assembleDebug` passes
2. **Room**: Entity + DAO + Database + AppModule → compile
3. **DI**: AppModule provides Database, DAOs, WorkManager, SyncPreferences
4. **Theme**: Theme.kt, Color.kt, Type.kt
5. **Application class + MainActivity** with NavHost → launches to placeholder
6. **ExpenseListScreen + ExpenseListViewModel** → empty state
7. **EntryScreen + EntryViewModel** → create/edit expenses locally
8. **SyncPreferences + GoogleSheetsService** → Sheets skeleton
9. **SyncWorker** + WorkManager periodic scheduling → sync to Sheets
10. **SettingsScreen** → Google account + spreadsheet ID config
11. **Polish**: error handling, loading states, empty states, delete
